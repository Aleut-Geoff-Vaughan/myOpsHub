# myForecast Portal - Design Document

## Executive Summary

This document outlines the redesign of the staffing/forecasting solution into a dedicated **myForecast** portal. The goal is to create a collaborative forecasting experience for PMs, Finance leads, P&L owners, and executives with improved usability, scalability, and oversight capabilities.

---

## 1. Current State Analysis

### 1.1 Existing Implementation

The current staffing/forecasting functionality is spread across multiple portals:

| Location | Features | Issues |
|----------|----------|--------|
| **Me Portal** (`/staffing`) | Personal assignments, capacity view, timeline | Good for individual view, but limited collaboration |
| **Manager Portal** (`/manager/staffing`) | Team staffing, 6-month timeline | No direct forecast editing capability |
| **Manager Portal** (`/manager/forecast-approvals`) | Approval workflow | Limited visibility into forecast health |
| **Admin Portal** (`/admin/staffing/*`) | Forecasts grid, versions, import/export | Too technical, buried in admin interface |

### 1.2 Key Problems

1. **Fragmented Experience**: Forecasting features scattered across portals
2. **Limited Collaboration**: No shared workspace for PM + Finance collaboration
3. **Poor Oversight**: Executives lack dashboard for forecast health/completeness
4. **Scalability Concerns**: Grid loads all assignments at once (performance)
5. **Workflow Gaps**: No clear PM â†’ Finance â†’ P&L Lead approval chain
6. **Missing Analytics**: No forecast effectiveness/accuracy tracking
7. **No Weekly Granularity**: Monthly-only forecasts insufficient for some projects
8. **No Budget Tracking**: Budget data not integrated with forecasts
9. **Manual Data Entry**: No bulk import for actuals or subcontractor forecasts

## 3. Design Decisions (Based on Requirements)

| Question | Decision | Impact |
|----------|----------|--------|
| **Forecast Granularity** | Monthly + Weekly | Add week field to Forecast entity; UI toggle for weekly vs monthly view |
| **Approval Chain** | PM â†’ Finance â†’ **P&L (Final)** | P&L has final approval authority; Finance recommends |
| **What-If Scenarios** | Both PM and Finance can create | Add version creation permission to PM role |
| **Subcontractor Forecasts** | Spreadsheet upload | Add import template and validation for subcontractor hours |
| **Actuals Integration** | ERP feed + Spreadsheet upload | Create Actuals entity; import endpoints; variance calculations |
| **Budget Tracking** | Yes - at WBS/Project level | Add budget fields to Project and WbsElement entities |
| **Notifications** | Deadlines + Approvals | Email/in-app for: upcoming deadlines, approval requests, rejections |
| **Mobile Access** | Individual: Yes, Team: Desktop only | Responsive design for personal views; desktop-optimized for grid editing |
